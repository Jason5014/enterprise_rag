# 企业RAG知识库系统 - 项目规划文档

## 一、项目概述

### 1.1 项目背景

基于 RAG Challenge 获奖方案的设计理念，搭建一个企业级知识库问答系统，支持对几家证券公司的内部文档（年报、报告、合同等）进行智能问答。

**参考项目**：RAG-Challenge-2（比赛获奖方案）

**改进方向**：
- 使用 MinerU 替代 Docling 进行 PDF 解析
- 配置化设计，支持多配置对比
- 自有 UI 界面（Streamlit）

### 1.2 项目目标

1. **数据处理**：将 PDF 文档解析、分块、索引，存入向量数据库
2. **智能问答**：支持混合检索、LLM 重排、Chain-of-Thought 答案生成
3. **可配置化**：关键参数可配置，支持多种配置组合对比
4. **用户友好**：Web UI 界面，支持对话式问答

## 四、功能模块设计

### 4.1 PDF解析模块（pdf_mineru.py）

**功能**：使用 MinerU 解析 PDF 为结构化 JSON

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `parse_tables` | bool | True | 是否解析表格 |
| `extract_images` | bool | False | 是否提取图片描述 |
| `page_range` | str | "all" | 解析页码范围 |
| `output_format` | str | "json" | 输出格式 |

**输出结构**：
```json
{
  "metainfo": {
    "company_name": "中芯国际",
    "sha1_name": "abc123",
    "total_pages": 100
  },
  "content": {
    "pages": [
      {
        "page": 1,
        "text": "页面文本内容",
        "tables": [
          {
            "table_id": 1,
            "html": "<table>...</table>",
            "page": 1
          }
        ]
      }
    ],
    "chunks": []
  }
}
```

### 4.2 表格序列化模块（table_serializer.py）

**功能**：将 HTML 表格转换为文本描述

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `serialization_model` | str | "qwen-turbo" | 序列化模型 |
| `batch_size` | int | 5 | 批量处理大小 |
| `max_workers` | int | 3 | 并行工作数 |

**输入**：HTML 表格字符串
**输出**：文本描述，如 `"产品: A产品 | 营收: 100万 | 增长: 20%"`

### 4.3 文本分块模块（text_splitter.py）

**功能**：将长文本分割为小块，支持父子chunk关联

**核心概念**：
- **父Chunk（Parent Chunk）**：完整的一段/页内容（如500-800字）
- **子Chunk（Child Chunk）**：从父Chunk切分的小块（如300字）
- 子Chunk通过 `parent_id` 关联父Chunk

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `chunk_size` | int | 500 | 子Chunk大小（字符数） |
| `chunk_overlap` | int | 150 | 子Chunk重叠大小 |
| `parent_chunk_size` | int | 800 | 父Chunk大小 |
| `separators` | list | ["\n\n", "\n", " "] | 分隔符列表 |
| `enable_parent_retrieval` | bool | True | 是否启用父子关联 |

**输出结构**：
```json
{
  "chunks": [
    {
      "chunk_id": "p1_c1",
      "parent_id": "p1_parent",
      "text": "子chunk文本内容...",
      "metadata": {
        "page": 1,
        "char_start": 0,
        "char_end": 500
      }
    }
  ],
  "parent_chunks": [
    {
      "parent_id": "p1_parent",
      "text": "父chunk完整内容（800字）...",
      "metadata": {
        "page": 1,
        "char_start": 0,
        "char_end": 800
      }
    }
  ]
}
```

**父子关联示意**：
```
父Chunk (800字)
┌──────────────────────────────────────────────────────────────┐
│ "中芯国际2024年营收500亿元，同比增长20%..."                 │
│ "公司在先进制程方面取得突破..."                              │
│ "展望2025年..."                                              │
└──────────────────────────────────────────────────────────────┘
        │
        │ 切分
        ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ 子Chunk 1   │  │ 子Chunk 2   │  │ 子Chunk 3   │
│ (300字)    │  │ (300字)    │  │ (200字)    │
│ parent_id= │  │ parent_id= │  │ parent_id= │
│ "p1_parent"│  │ "p1_parent"│  │ "p1_parent"│
└─────────────┘  └─────────────┘  └─────────────┘
```

### 4.7 重排器模块（reranker.py）

**功能**：使用 LLM 对检索结果重排，**重排时携带父chunk完整上下文**

**核心逻辑**：
1. 子Chunk匹配检索 → 获取子Chunk的 `parent_id`
2. 拼接父Chunk完整内容（可能包含多个相关子Chunk的上下文）
3. 将父Chunk内容发给 LLM 进行重排打分

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `rerank_model` | str | "qwen-turbo" | 重排模型 |
| `rerank_top_k` | int | 5 | 重排后返回数量 |
| `llm_weight` | float | 0.7 | LLM分数权重 |
| `batch_size` | int | 2 | 批量大小 |
| `max_workers` | int | 1 | 并行工作数（串行避免QPS超限）|

**重排时父子Chunk拼接示意**：
```
检索结果：
[
  { "chunk_id": "p1_c1", "parent_id": "p1_parent", "text": "子chunk内容..." },
  { "chunk_id": "p2_c3", "parent_id": "p2_parent", "text": "子chunk内容..." }
]

重排输入构建：
┌──────────────────────────────────────────────────────────────┐
│  [Chunk p1_c1 所属父Chunk]                                  │
│  "中芯国际2024年营收500亿元..."                              │
│  "公司在先进制程方面取得突破..."  ← 完整上下文              │
│  "展望2025年..."                                              │
├────────────────────────────────────────────────────────────┤
│  [Chunk p2_c3 所属父Chunk]                                  │
│  "公司员工总数2万人..."                                      │
│  "研发投入占比15%..."                                         │
└────────────────────────────────────────────────────────────┘

↓ 批量发送给 LLM 重排打分

combined_score = 0.7 × LLM_score + 0.3 × vector_distance
```

**说明**：重排时使用父Chunk而不是子Chunk，可以获得更完整的上下文信息，提高重排准确性。

### 4.9 答案生成模块（answer_generator.py）

**功能**：基于检索上下文（携带父chunk完整上下文）生成 Chain-of-Thought 答案

**核心逻辑**：
1. 检索阶段：子Chunk匹配 → 获取父Chunk完整内容
2. 重排阶段：基于父Chunk打分排序
3. 生成阶段：**使用父Chunk完整上下文**生成答案，而非子Chunk

**输入构建**：
```
重排后的Top-K结果 → 获取每个结果的父Chunk → 拼接完整上下文

┌─────────────────────────────────────────────────────────────┐
│  [父Chunk 1 - 完整上下文]                                   │
│  "中芯国际2024年营收500亿元..."                              │
│  "公司在先进制程方面取得突破..."                            │
│  "展望2025年..."                                            │
├─────────────────────────────────────────────────────────────┤
│  [父Chunk 2 - 完整上下文]                                   │
│  "公司员工总数2万人..."                                      │
│  "研发投入占比15%..."                                       │
└─────────────────────────────────────────────────────────────┘
                          ↓
              构建 RAG 上下文 + 问题 → LLM 生成答案
```

**输出结构**：
```python
{
    "step_by_step_analysis": "详细推理过程（5步+, 150字+）",
    "reasoning_summary": "50字以内的摘要",
    "relevant_pages": [1, 3, 5],  # 引用页码
    "final_answer": "具体答案或 'N/A'",
    "used_parent_chunks": ["p1_parent", "p2_parent"]  # 使用的父Chunk ID列表
}
```

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `answer_model` | str | "qwen-turbo" | 答案生成模型 |
| `temperature` | float | 0.1 | 生成温度 |
| `max_tokens` | int | 2000 | 最大token数 |

**设计理由**：
- 子Chunk检索精准但可能上下文不完整
- 父Chunk包含更完整的语义信息
- 答案生成时使用父Chunk可以提高答案质量和一致性

### 4.11 MultiQuery 查询扩展模块（multi_query.py）

**功能**：将单一问题扩展为多个查询变体，改善检索覆盖率

**核心逻辑**：
1. 用户输入原始问题
2. LLM 生成 3-5 个语义相似的查询变体
3. 每个变体独立执行混合检索
4. 合并去重后得到最终候选集

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | True | 是否启用 |
| `num_variants` | int | 3 | 查询变体数量 |
| `model` | str | "qwen-turbo" | 扩展模型 |
| `include_original` | bool | True | 是否保留原始查询 |

**输出结构**：
```python
{
    "original_query": "中芯国际2024年营收是多少？",
    "query_variants": [
        "中芯国际2024年营业收入",
        "SMIC 2024年财报收入",
        "中芯国际年度营收数据"
    ],
    "merged_candidates": [...]  # 合并后的检索结果
}
```

**MultiQuery 流程示意**：
```
用户问题: "中芯国际2024年营收是多少？"
                │
                ▼
        ┌───────────────┐
        │  LLM 生成变体  │
        └───────────────┘
                │
        ├──→ "中芯国际2024年营业收入"
        ├──→ "SMIC 2024年财报收入"
        ├──→ "中芯国际年度营收数据"
        └──→ "中芯国际2024年财务业绩"
                │
                ▼
    ┌─────────────────────────┐
    │  并行执行混合检索        │
    │  BM25 + Vector         │
    └─────────────────────────┘
                │
                ▼
        结果合并 + 去重
                │
                ▼
        Top-K 候选结果
```

### 4.12 Query Rewriter 查询改写模块（query_rewriter.py）

**功能**：将用户口语化问题改写为检索友好的查询表达式

**改写策略**：
| 场景 | 原始问题 | 改写后 |
|------|---------|--------|
| 口语化 | "我想知道公司去年赚了多少钱" | "公司2024年营业收入" |
| 指代消解 | "它的研发投入是多少" | "中芯国际2024年研发投入" |
| 缩写展开 | "华为的营收" | "华为技术有限公司营业收入" |
| 同义词 | "公司员工多少人" | "公司员工总数" |

**配置项**：
| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | True | 是否启用 |
| `model` | str | "qwen-turbo" | 改写模型 |
| `preserve_original` | bool | True | 是否保留原始问题用于对比 |

**输出结构**：
```python
{
    "original": "我想知道公司去年赚了多少钱",
    "rewritten": "中芯国际2024年营业收入",
    "history_context": "用户问的是中芯国际，不是其他公司",  # 可选的对话历史
    "confidence": 0.92
}
```

### 4.13 Evaluator 评估模块（evaluator.py）

**功能**：评估问答质量，生成评估指标和改进建议

**评估维度**：
| 维度 | 指标 | 说明 |
|------|------|------|
| 准确性 | Exact Match, F1 | 答案与标准答案的匹配度 |
| 相关性 | Context Precision | 检索内容与问题的相关程度 |
| 完整性 | Coverage Rate | 答案覆盖问题要点的比例 |
| 可解释性 | Citation Accuracy | 引用的页码是否正确 |

**子模块**：

#### 4.13.1 QA生成器（qa_generator.py）
- 从文档中自动生成问答对
- 支持多种问题类型（事实型、比较型、推理型）
- 用于测试集构建和评估

#### 4.13.2 检索评估器（retrieval_evaluator.py）

```python
from typing import List, Optional

@dataclass
class BaseConfig:
    """配置基类，所有配置继承此类"""

    def to_dict(self) -> dict:
        return {k: v for k, v in self.__dict__.items() if not k.startswith('_')}

    @classmethod
    def from_dict(cls, d: dict):
        return cls(**d)

    def validate(self) -> bool:
        """验证配置合法性"""
        raise NotImplementedError
```
