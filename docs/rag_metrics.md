## 一、检索质量指标

### 1.1 Recall@K（召回率）

**定义**：前K个检索结果中包含的相关文档比例

**公式**：
```
Recall@K = |检索到的相关文档| / |所有相关文档|
```

**示例**：
```
相关文档: [A, B, C, D]
检索结果 Top-5: [A, C, E, F, B]
Recall@5 = 3/4 = 0.75
```

**意义**：衡量检索的"查全率"，是否有遗漏

**优化方向**：
- Recall@5 低 → 增加 top_k、启用 MultiQuery
- Recall@10 高但 Recall@5 低 → 启用重排

---

### 1.2 Precision@K（精确率）

**定义**：前K个检索结果中相关文档的比例

**公式**：
```
Precision@K = |检索到的相关文档| / K
```

**示例**：
```
检索结果 Top-5: [A, C, E, F, B]
其中相关: [A, C, B]
Precision@5 = 3/5 = 0.6
```

**意义**：衡量检索的"查准率"，减少噪音

**优化方向**：
- Precision 低 → 调整相似度阈值、优化 Embedding

---

### 1.3 MRR（平均倒排名）

**定义**：第一个相关结果的排名倒数

**公式**：
```
MRR = 1 / rank_of_first_relevant_result
```

**示例**：
```
检索结果: [E, F, A, B, C]
第一个相关结果 A 在第3位
MRR = 1/3 = 0.33
```

**意义**：衡量用户找到答案的效率

**优化方向**：
- MRR 低 → 加强重排、调整 LLM 权重

---

### 1.4 NDCG@K（归一化折损累积增益）

**定义**：考虑排序位置的检索质量指标

**公式**：
```
DCG@K = Σ(rel_i / log2(i+1))
NDCG@K = DCG@K / IDCG@K
```

**示例**：
```
检索结果: [相关, 不相关, 相关, 相关, 不相关]
DCG@5 = 1/log2(2) + 0 + 1/log2(4) + 1/log2(5) + 0
      = 1 + 0 + 0.5 + 0.43 = 1.93
IDCG@5 = 1 + 0.63 + 0.5 + 0.43 + 0.39 = 2.95
NDCG@5 = 1.93/2.95 = 0.65
```

**意义**：综合考虑相关性和排序位置

**优化方向**：
- NDCG 低 → 优化 Embedding 模型、调整重排策略

---

### 1.5 MAP（平均精确率均值）

**定义**：所有相关文档的精确率平均值

**公式**：
```
AP = Σ(Precision@k × rel(k)) / |相关文档数|
MAP = 所有查询的 AP 均值
```

**意义**：综合评估检索系统的整体表现

---

### 1.6 Hit@K（命中率）

**定义**：Top-K 结果中是否至少命中一个预期 chunk（二值指标）

**公式**：
```
Hit@K = 1 if |Top-K ∩ relevant| > 0 else 0
```

**示例**：
```
预期 chunk: [A, B, C]
Top-1: [X]           → Hit@1 = 0
Top-3: [X, A, Y]     → Hit@3 = 1
Top-5: [X, Y, Z, W, V] → Hit@5 = 0
```

**意义**：最直观的检索质量指标。Recall@K 受预期 chunk 数量影响（如 5 个预期 chunk 时 Recall@1 最高 20%），Hit@K 不受影响。

**与 Recall@K 的区别**：
| 指标 | 公式 | 特点 |
|------|------|------|
| Recall@K | 命中数/预期总数 | 衡量"覆盖了多少"，受预期数量影响 |
| Hit@K | 命中>0?1:0 | 衡量"是否找到"，不受预期数量影响 |

**推荐用法**：
- Hit@1: 第一个结果是否正确（用户体验核心指标）
- Hit@5: 前5个结果是否包含答案（检索质量核心指标）
- Recall@5: 配合 Hit@5 看覆盖率

## 二、生成质量指标

### 2.1 Faithfulness（忠实度）

**定义**：答案是否基于检索到的上下文，不包含幻觉

**评估方法**：
```
答案: "中芯国际2024年营收523亿元"
上下文: "中芯国际2024年营业收入为523.48亿元"

忠实度 = 1.0（答案有依据）
```

**评分标准**：
| 分数 | 含义 |
|------|------|
| 1.0 | 完全基于上下文 |
| 0.7 | 大部分基于上下文，有少量推断 |
| 0.3 | 部分基于上下文，有明显推断 |
| 0.0 | 完全幻觉，无依据 |

---

### 2.2 Answer Relevance（答案相关性）

**定义**：答案是否与问题相关

**评估方法**：
```
问题: "中芯国际2024年营收是多少？"
答案A: "523亿元" → 相关性 1.0
答案B: "公司总部在上海" → 相关性 0.0
```

**评分标准**：
| 分数 | 含义 |
|------|------|
| 1.0 | 完全回答了问题 |
| 0.7 | 部分回答了问题 |
| 0.3 | 与问题相关但未回答 |
| 0.0 | 完全不相关 |

---

### 2.3 Completeness（完整性）

**定义**：答案是否完整回答了问题的所有方面

**示例**：
```
问题: "公司2024年的营收和利润分别是多少？"
答案A: "营收523亿元，利润85亿元" → 完整性 1.0
答案B: "营收523亿元" → 完整性 0.5
```

---

### 2.4 Fluency（流畅度）

**定义**：答案的语言是否通顺、专业

**评分标准**：
| 分数 | 含义 |
|------|------|
| 1.0 | 语言流畅、专业 |
| 0.7 | 基本流畅，偶有不通顺 |
| 0.3 | 语言生硬，有多处不通顺 |
| 0.0 | 难以理解 |

---

## 三、端到端指标

### 3.1 Exact Match（精确匹配）

**定义**：答案与标准答案完全一致

**适用场景**：事实型问题（数字、名称、日期）

**示例**：
```
标准答案: "523.48亿元"
预测答案: "523.48亿元" → EM = 1
预测答案: "约523亿元" → EM = 0
```

---

### 3.2 F1 Score

**定义**：答案与标准答案的词级别重叠度

**公式**：
```
Precision = |预测词 ∩ 标准词| / |预测词|
Recall = |预测词 ∩ 标准词| / |标准词|
F1 = 2 × P × R / (P + R)
```

**适用场景**：长文本答案

---

### 3.3 LLM-as-Judge

**定义**：使用 LLM 评估答案质量

**评估维度**：
- 正确性（Correctness）
- 有用性（Helpfulness）
- 完整性（Completeness）
- 无害性（Harmlessness）

**Prompt 示例**：
```
请评估以下答案的质量：

问题：{question}
参考答案：{reference}
预测答案：{prediction}

请从以下维度评分（1-5分）：
1. 正确性：答案是否准确
2. 有用性：答案是否有帮助
3. 完整性：答案是否完整

输出 JSON 格式：
{"correctness": 4, "helpfulness": 5, "completeness": 3}
```

---

### 3.4 Citation Accuracy（引用准确率）

**定义**：答案中的引用是否正确

**评估内容**：
- 引用的页码是否存在
- 引用的内容是否支持答案
- 是否有遗漏引用

## 四、系统性能指标

### 4.1 Latency（延迟）

**分阶段统计**：
```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ Query       │ Retrieval   │ Rerank      │ Generation  │
│ Rewrite     │             │             │             │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ 200ms       │ 500ms       │ 1000ms      │ 800ms       │
└─────────────┴─────────────┴─────────────┴─────────────┘
总延迟: 2500ms
```

**目标**：
- 首字延迟（TTFT）< 500ms
- 总延迟 < 3000ms

---

### 4.2 Throughput（吞吐量）

**定义**：单位时间处理的查询数

**公式**：
```
Throughput = 查询数 / 时间（QPS）
```

---

### 4.3 Cost（成本）

**分项统计**：
```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ Embedding   │ LLM Query   │ LLM Rerank  │ LLM Answer  │
│             │ Rewrite     │             │             │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ $0.001      │ $0.002      │ $0.005      │ $0.01       │
└─────────────┴─────────────┴─────────────┴─────────────┘
单次查询成本: $0.018
```

## 五、高级评估方法

### 5.1 人工评估

**评估维度**：
| 维度 | 说明 | 评分 |
|------|------|------|
| 准确性 | 答案是否正确 | 1-5 |
| 相关性 | 答案是否切题 | 1-5 |
| 完整性 | 答案是否全面 | 1-5 |
| 可读性 | 答案是否易读 | 1-5 |
| 引用 | 引用是否准确 | 1-5 |

---

### 5.2 自动评估框架

**RAGAS 框架**：
```python
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy, context_precision

result = evaluate(
    dataset=eval_dataset,
    metrics=[faithfulness, answer_relevancy, context_precision]
)
```

**指标**：
- `faithfulness`：忠实度
- `answer_relevancy`：答案相关性
- `context_precision`：上下文精确率
- `context_recall`：上下文召回率

---

### 5.3 A/B 测试

**流程**：
```
1. 准备测试集（100+ 问题）
2. 配置A vs 配置B
3. 收集指标
4. 统计显著性检验
5. 选择更优配置
```

## 六、指标选择建议

### 按场景选择

| 场景 | 重点指标 | 原因 |
|------|----------|------|
| 事实问答 | Recall, EM, F1 | 需要准确找到答案 |
| 开放问答 | Faithfulness, Relevance | 需要生成有用答案 |
| 摘要任务 | Completeness, Fluency | 需要全面且流畅 |
| 对话系统 | Latency, Relevance | 需要快速响应 |

### 按优先级选择

**必须指标**（第一梯队）：
- Hit@5：检索是否能找到（推荐，最直观）
- Hit@1：第一个结果是否正确
- MRR：找到的效率
- Faithfulness：是否有幻觉

**重要指标**（第二梯队）：
- Precision@5：噪音多不多
- Answer Relevance：答案是否切题
- Latency：响应速度

**可选指标**（第三梯队）：
- NDCG：排序质量
- Completeness：答案完整性
- Cost：成本控制
